## Featured Work

## 1) PEA
### AI-Assisted Mobile Exercise Analysis App

**PEA** is a Flutter-based fitness analysis app focused on live camera input, pose detection, rep counting, session tracking, and guided workout support.

Main focus areas:

- real-time pose landmark analysis
- repetition counting
- movement phase tracking
- basic scoring and feedback generation
- session history and summaries
- Firebase-backed mobile product flow

This is the project that best represents my product direction: **AI-assisted mobile experiences that are designed like real apps, not just experiments**.

---

## 2) Drowsiness Detector
### Android-Based Driver Safety & Incident Tracking App

This project is an Android application that uses front-camera analysis to detect signs such as:

- drowsiness
- yawning
- distraction
- fixation / loss of attention

It combines:

- CameraX
- ML Kit Face Detection
- local incident storage
- background sync
- map and location features
- fleet / incident monitoring screens

What makes it important to me is that it goes beyond detection logic and becomes a more complete mobile system with **alerts, offline-first incident flow, and operational screens**.

---

## 3) server
### Spring Boot Backend for Mobile Safety Workflows

This backend powers the incident / fleet side of the driver safety system.

Main responsibilities:

- driver and manager registration
- login flow
- incident reporting
- driver-based incident history
- group / fleet creation and membership actions
- fleet-wide incident and driver listing

Tech used here includes:

- Spring Boot
- Spring Web MVC
- Spring Data JPA
- PostgreSQL
- Docker

This repo represents the backend side of my work: not just building mobile UIs, but also supporting them with APIs and persistence layers.
